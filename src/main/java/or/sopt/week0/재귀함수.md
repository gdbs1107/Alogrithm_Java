#  재귀함수
- 재귀함수는 정의 단계에서 본인을 재참조하는 함수를 의미한다
- 전달되는 상태인 매개변수가 달라질 뿐, 똑같은 일을 하는 함수
- 큰 문제를 작은 부분 문제로 나눠서 풀 때 사용한다

# 주의사항
- 반드시 **종료조건**이 필요하다
- 사이클이 있다면 쓰면 안된다
- 반복문이 될 것 같으면 **그냥 반복문** 써라 -> 효율적인 측면의 문. 보통의 경우에서 반복문이 빠르다
- 점화식을 수립하고 이를 코드로 구현하라
- 디버깅을 자주하면 구현에 좋을 것

-> 함수호출이 매우 많이 되고, 중복 호출도 매우 많아서 성능 상의 문제가 존재한다
-> 그래서 나온게 DP인데, 그건 나중에 알아보도록 하자

# 재귀함수가 불리한 이유

재귀함수는 기본적으로 스택을 이용하여 depth를 쌓게 된다.
그러니까 재귀가 깊어질수록 JVM의 memory 사용량이 늘어나게 되고, 이는 곧 StackOverFlow 에러를 터뜨릴 수 있다.

그리고 사실 메모리를 크게 점유하는 로직이라는 부분에서부터 이미 불리한 구현 방식임.
